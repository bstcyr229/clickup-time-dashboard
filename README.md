# ClickUp Time Intelligence Dashboard

A Python + Streamlit app that turns raw time tracking data into a visual dashboard — showing team workload, estimated vs. actual hour variance, and billable cost by assignee, project, and task.

---

## What It Does

Most time tracking tools show you what was logged. This dashboard shows you what it *means*. You get three views in one app: how many hours each team worked per day of the week, how individual assignees are tracking against their estimates, and a task-level breakdown of where time actually went. Everything is displayed as interactive charts and metrics — no spreadsheets, no manual exports.

---

## Why It Exists

ClickUp's native reporting can tell you hours logged. It can't easily tell you which hours were billable, whether your estimates were accurate, or which team member is consistently over capacity. Operations teams and project managers end up exporting data and reconciling it manually — a process that can take hours every week and still produces a number you're not fully confident in. This dashboard automates that reconciliation and makes the answer visible in seconds.

## Views

## How to Run It Locally

**Requirements:** Python 3.9+

```bash
# 1. Clone the repo
git clone https://github.com/YOUR_USERNAME/time-tracker.git
cd time-tracker

# 2. Create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate        # Mac/Linux
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Launch the app
streamlit run main.py
```

The app will open automatically in your browser at `http://localhost:8501`.

## Data Note

The app currently runs on **dummy data** generated by `dummy_data.py`. This is intentional — it lets anyone clone the repo and see a fully functional dashboard without needing a ClickUp account or API key.

The dummy data simulates a realistic one week work period across seven team members, three teams, and four projects. Hour estimates and actuals are generated using randomized variance to reflect real-world patterns (teams over capacity, estimation drift by assignee, etc.).

**A note on the "Team" grouping:** ClickUp's API exposes teams as User Groups,
the API terminology differs from the UI. In a live version, the dashboard would 
pull User Groups directly from the API to define the team dimension. The current 
version uses dummy data with manually assigned teams as ClickUp at this time does not allow users to create dummy users.
